# Code Ownership Map

This document maps application source code to features defined in **FEATURE_REGISTRY.md**. It is generated by analyzing the repository. Cursor must read this document before modifying the codebase.

---

## Cursor Modification Rules

1. **Files mapped to implemented features are stable modules.** Cursor must NOT refactor or rename these modules.
2. Cursor must NOT modify stable modules unless explicitly instructed.
3. New features should create new service modules when possible.
4. Existing services must only be modified when explicitly instructed.
5. Always read **FEATURE_REGISTRY.md** and **CODE_OWNERSHIP_MAP.md** before making changes.

---

## Architecture Layers

| Layer | Path | Purpose |
|-------|------|---------|
| **Frontend** | `frontend/src/app/`, `frontend/src/components/` | Next.js pages, UI components |
| **Backend API** | `backend/app/api/` | FastAPI routers, HTTP endpoints |
| **Backend Services** | `backend/app/services/` | Business logic, monitoring, pipelines |
| **Workers** | `backend/app/services/*_worker.py` | Async/periodic jobs (media, sentiment, topic, social, reddit, youtube) |
| **Core** | `backend/app/core/` | Config loaders, health, logging, hash utils, indexes |
| **Configuration** | `config/*.yaml` | Environment and feature config |
| **Database Collections** | MongoDB: `media_articles`, `social_posts`, `conversations`, `messages` | Persisted data per feature |
